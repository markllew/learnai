Module 3: Functions

Creating Reusable and Organized Blocks of Code

Imagine writing a large program where you need to perform the same task over and over again. You could copy and paste the code, but that's inefficient and makes your code hard to update. Functions solve this problem by letting you package a block of code, give it a name, and call it whenever you need it. This is a core principle of programming known as DRY.

1. Defining and Calling a Function

You define a function using the def keyword, followed by the function's name, parentheses (), and a colon :. The code inside the function must be indented. A docstring is a good practice for explaining your function's purpose.


# Define a simple function
def greet_user():
    """This is a docstring. It explains what the function does."""
    print("Hello and welcome to the program!")

# Call the function to execute its code
greet_user()
greet_user()

2. Parameters and Arguments

Functions become much more powerful when you can pass information into them. A parameter is the variable placeholder in the function definition, and an argument is the actual value you pass in when you call the function.


def greet_by_name(name, greeting="Hello"): # 'greeting' has a default value
    """Greets a user with a specific name and optional greeting."""
    print(f"{greeting}, {name}!")

# Call the function with different arguments
greet_by_name("Alice") # Uses the default greeting
greet_by_name("Bob", "Good morning") # Overrides the default

3. Return Values

Functions can also process information and send a result back to the code that called them using the return keyword. This allows you to use the output of a function in other parts of your code.


def calculate_area(width, height):
    """Calculates the area of a rectangle and returns it."""
    if width < 0 or height < 0:
        return None # Return a special value if input is invalid
    return width * height

# Call the function and store the result in a variable
rect_area = calculate_area(10, 5)
if rect_area is not None:
    print(f"The area is: {rect_area}")
    print(f"The area of a second rectangle is: {calculate_area(3, 7)}")
else:
    print("Invalid input for width or height.")

AI-Proof Exercise: The Text Analyzer

Write a function called analyze_text that takes one argument: a string of text.

This function should return a dictionary where the keys are the unique words in the text and the values are the number of times each word appeared. The function should be case-insensitive and should ignore basic punctuation (.,!?).

Challenge: Break your logic into smaller helper functions. For example, you could have one function clean_text(raw_string) that takes the raw string and returns a clean list of words, and your main analyze_text function could then call this helper function before counting the words. This practice of breaking down complex problems is crucial in programming.

Ethical Spotlight: Algorithmic Bias in Functions

Imagine a function is_professional(text) designed to scan emails for a hiring platform. If the logic inside this function was built on data where certain dialects or names were disproportionately marked as "unprofessional," the function would learn and perpetuate this bias. Even a small, seemingly simple function can have significant ethical implications if its internal logic is flawed or built on biased data.
